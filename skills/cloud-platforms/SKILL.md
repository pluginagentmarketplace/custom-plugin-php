# Cloud Platforms Skill

Master cloud platforms and cloud-native architecture.

## Amazon Web Services (AWS)

### Core Services
- **EC2** - Virtual machines
- **S3** - Object storage
- **RDS** - Managed relational databases
- **DynamoDB** - NoSQL database
- **Lambda** - Serverless compute

### Developer Tools
- **CodePipeline** - CI/CD automation
- **CodeBuild** - Build service
- **CodeDeploy** - Deployment automation
- **CloudFormation** - Infrastructure as code

### Networking
- **VPC** - Virtual private cloud
- **CloudFront** - Content delivery network
- **Route 53** - DNS service
- **Elastic Load Balancing** - Load distribution

### Monitoring
- **CloudWatch** - Metrics, logs, monitoring
- **X-Ray** - Distributed tracing

## Google Cloud Platform (GCP)

### Compute
- **Compute Engine** - VMs
- **Cloud Run** - Serverless containers
- **Google Kubernetes Engine (GKE)** - Managed Kubernetes

### Data & Analytics
- **BigQuery** - Data warehouse
- **Dataflow** - Stream/batch processing
- **Pub/Sub** - Messaging service

### Storage
- **Cloud Storage** - Object storage
- **Cloud Firestore** - NoSQL database

## Microsoft Azure

### Compute
- **Virtual Machines** - IaaS
- **App Service** - PaaS for web apps
- **Azure Kubernetes Service (AKS)** - Managed Kubernetes

### Data
- **SQL Database** - Managed SQL
- **Cosmos DB** - NoSQL multi-model
- **Synapse Analytics** - Data warehouse

### Developer Tools
- **Azure DevOps** - CI/CD, project management
- **Azure Pipelines** - Build and release

## Serverless Computing

### Functions as a Service
- **AWS Lambda**
- **Google Cloud Functions**
- **Azure Functions**

### Benefits
- No server management
- Auto-scaling
- Pay per execution
- Event-driven

### Best For
- Microservices
- Event handling
- API backends
- Scheduled tasks

## Containerization

### Docker
- Container images
- Registry (Docker Hub, ECR, GCR)
- Docker Compose

### Kubernetes (K8s)
- Pod orchestration
- Service management
- ConfigMaps and Secrets
- StatefulSets, DaemonSets

### Managed Services
- **AWS EKS** - Elastic Kubernetes Service
- **Google GKE** - Google Kubernetes Engine
- **Azure AKS** - Azure Kubernetes Service

## Infrastructure as Code

### Terraform
- Multi-cloud support
- HCL language
- State management
- Modules and workspaces

### Native IaC
- **AWS CloudFormation** - JSON/YAML
- **Azure Resource Manager** - JSON templates
- **Google Deployment Manager** - YAML

## Recommended Learning Path

1. Choose one cloud provider (AWS recommended for breadth)
2. Master core services (compute, storage, databases)
3. Learn infrastructure as code (Terraform)
4. Explore containerization (Docker, Kubernetes)
5. Master CI/CD and DevOps practices
6. Explore serverless and advanced services

## Key Concepts

- **Regions and Availability Zones** - Geographical distribution
- **Networking** - VPCs, security groups, load balancing
- **IAM** - Identity and access management
- **Cost Management** - Reserved instances, spot instances
- **Auto-scaling** - Dynamic resource allocation
- **Disaster Recovery** - RTO, RPO, backup strategies
