- **REST APIs** - Resource design, HTTP methods, status codes
- **GraphQL** - Query language, resolvers, subscriptions
- **gRPC** - Binary protocol, streaming, performance
- **SOAP & WebServices** - Legacy integration
- **API Security** - JWT, OAuth2, API keys
- **Documentation** - OpenAPI, Swagger

### 4. Database Technologies

**Relational (SQL):**
- PostgreSQL (recommended)
- MySQL, MariaDB
- Microsoft SQL Server
- Database design, normalization, indexes

**NoSQL:**
- MongoDB (document)
- Cassandra (wide-column)
- Neo4j (graph)
- DynamoDB (AWS)

**Caching:**
- Redis (in-memory data store)
- Memcached
- Cache strategies (cache-aside, write-through)

**Vector Databases (Emerging):**
- Pinecone, Chroma, Milvus

### 5. Architecture Patterns
- **Monolithic** - Single deployment unit
- **Microservices** - Distributed, independently deployable services
- **Serverless** - Event-driven, pay-per-use
- **Event-Driven** - Async communication
- **CQRS** - Command Query Responsibility Segregation
- **Saga Pattern** - Distributed transactions

### 6. Infrastructure & DevOps
- Containerization (Docker)
- Orchestration (Kubernetes, Docker Compose)
- Cloud platforms (AWS, GCP, Azure)
- CI/CD pipelines (GitHub Actions, Jenkins)
- Monitoring and logging

### 7. Security
- Authentication (JWT, OAuth2, sessions)
- Authorization and RBAC
- Data encryption (TLS, encryption at rest)
- SQL injection prevention
- CORS and security headers
- API rate limiting

### 8. Testing
- Unit testing frameworks
- Integration testing
- API testing (Postman, API testing tools)
- Load testing
- Contract testing
