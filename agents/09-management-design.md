### 1. Product Management

**Product Strategy:**
- Vision and mission definition
- Market analysis and competitive landscape
- Target audience identification
- Value proposition development
- Roadmap planning and prioritization
- OKRs (Objectives and Key Results)

**Discovery & Research:**
- User interviews and user research
- Personas and user journeys
- Jobs to be Done framework
- Jobs Stories
- Feature prioritization frameworks
  - RICE (Reach, Impact, Confidence, Effort)
  - MoSCoW (Must, Should, Could, Won't)
  - Kano Model

**Execution & Delivery:**
- Agile methodologies (Scrum, Kanban)
- Sprint planning
- User story writing
- A/B testing and experimentation
- Release management
- Beta programs and early access

**Metrics & Analytics:**
- Key Performance Indicators (KPIs)
- Customer Acquisition Cost (CAC)
- Lifetime Value (LTV)
- Retention and churn analysis
- Funnel analysis
- Cohort analysis
- Net Promoter Score (NPS)
- Conversion optimization

**Product Leadership:**
- Stakeholder management
- Cross-functional collaboration
- Communication and storytelling
- Data-driven decision making
- Handling trade-offs
- Vision communication

**Tools:**
- Jira, Linear (project management)
- Figma (design collaboration)
- Amplitude, Mixpanel, Heap (analytics)
- Intercom, Slack (communication)
- UserTesting, Typeform (research)

### 2. UX/UI Design

**User Experience (UX):**
- User research methodologies
- Personas and user journeys
- Wireframing and prototyping
- Information architecture
- Usability testing
- Accessibility (WCAG compliance)
- User psychology and behavior

**User Interface (UI):**
- Visual design principles
- Typography
- Color theory and palettes
- Layout and spacing
- Interaction design
- Micro-interactions
- Animation and motion

**Design Tools:**
- **Figma** (modern, collaborative, recommended)
- Adobe XD, Sketch
- Prototype tools: InVision, Framer
- Handoff tools: Zeplin, Figma Handoff

**Design Systems:**
- Component libraries
- Design tokens
- Pattern documentation
- Brand guidelines
- Scaling design across teams

**Research & Testing:**
- User interviews
- Surveys and questionnaires
- Usability testing
- A/B testing
- Accessibility testing
- Unmoderated testing

**Behavioral Design:**
- BJ Fogg's Behavior Model
- Habit formation (Hook Model)
- Nudge theory
- Loss aversion
- Social proof
- SCARCITY and urgency

**Interaction Design:**
- Microinteractions
- Animations and transitions
- Loading states
- Error handling
- Feedback mechanisms
- Gesture design (mobile)

### 4. Technical Writing

**Documentation Types:**
- API documentation
- User guides and tutorials
- Release notes and changelogs
- Architecture decision records (ADRs)
- Technical specifications
- How-to guides and FAQs
- Troubleshooting guides

**Writing Principles:**
- Clear and concise language
- Active voice
- Audience-appropriate tone
- Consistent terminology
- Proper structure and hierarchy
- Visual aids and diagrams
- Examples and code samples

**Documentation Tools:**
- Markdown (universal standard)
- Sphinx (Python docs)
- MkDocs, Docusaurus (documentation sites)
- Confluence, Notion (wikis)
- GitHub Pages (free hosting)
- ReadTheDocs (versioned docs)

**Best Practices:**
- Keep docs with code (git versioning)
- Regular updates and maintenance
- Feedback mechanism
- SEO optimization
- Localization support
- Accessibility compliance
